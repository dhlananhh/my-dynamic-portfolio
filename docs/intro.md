# Introduction

Welcome to the documentation for the **Dynamic Portfolio Project**.  
This document serves as a comprehensive entry point, explaining the purpose, scope, and structure of the project.

---

## Purpose

The portfolio was created to:
- Showcase my skills, projects, and professional experiences.
- Act as a central hub for recruiters, collaborators, and clients.
- Serve as a living knowledge base documenting the entire development journey.
- Provide transparency into the technical stack and design decisions.

---

## Scope of Documentation

This documentation covers:
- **About**: Background and vision of the portfolio.
- **Setup**: Installation and configuration instructions.
- **Architecture**: Tech stack, directory structure, and data flow.
- **Development**: Coding standards, workflows, and best practices.
- **Deployment**: Publishing strategy and CI/CD pipelines.
- **Testing**: Unit and end-to-end testing practices.
- **API Reference**: Payload CMS collections and endpoints.

---

## Tech Stack Overview

- **Frontend**: Next.js (React + TypeScript, App Router)
- **Styling**: CSS, SCSS, TailwindCSS
- **Content Management**: Payload CMS
- **Documentation**: MDX integrated directly into the portfolio
- **Testing**: Vitest + Playwright
- **Deployment**: Vercel with automated CI/CD
- **Internationalization**: `next-intl` for multi-language support

---

## Audience

This documentation is intended for:
- **Developers**: To understand the technical implementation and contribute.
- **Recruiters/Clients**: To explore the engineering practices behind the portfolio.
- **Future Me**: As a reference for maintenance and future improvements.

---

## Vision

The portfolio is not just a static showcase.  
It is a **dynamic knowledge base** that grows with my career, projects, and experiences.  
By embedding documentation directly into the site, I ensure that it reflects both my technical skills and my commitment to clear communication.
